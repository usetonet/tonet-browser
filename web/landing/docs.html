Documentation

Short guides for users and maintainers.

Installation

See the downloads section. Windows offers MSI and EXE; Linux supports .deb and a user install script; macOS is local builds until a signed bundle ships.

In-browser updates

Tonet checks the GitHub Releases API (it does not install binaries for you). By default it checks on startup. In Settings (⚙) you can choose:

  • On startup only — one check when the app opens.
  • Automatic (24 h) — repeats while Tonet is running.
  • Manual only — only with “Check now”.

Settings are stored in the system config directory (for example %APPDATA%\tonet on Windows).

Authenticode signing (Windows)

For SmartScreen trust, sign MSI and EXE with a code-signing certificate (SHA-256) on Windows using signtool from the Windows SDK. 

signtool sign /fd SHA256 /a /tr http://timestamp.digicert.com /td SHA256 Tonet-x.x.x-x64.msi
signtool sign /fd SHA256 /a /tr http://timestamp.digicert.com /td SHA256 Tonet-Setup-x.x.x-x64.exe

In CI, store the PFX in an encrypted secret and use a self-hosted runner or an action that invokes signtool. Never commit certificates to the repository.

Debian package (.deb)

The script packaging/debian/build-deb.sh builds a .deb from cargo build --release -p tonet. Requires dpkg-deb (Linux). 

cargo build --release -p tonet
./packaging/debian/build-deb.sh

AppImage (Linux)

Shipping a universal AppImage requires bundling GTK/GL dependencies — consider linuxdeploy + the GTK plugin once the binary stabilizes. A dedicated workflow may land in a future release.

Deploy this landing (Cloudflare Workers)

From the web/landing directory:

npm ci
npm run deploy

Set CLOUDFLARE_API_TOKEN and CLOUDFLARE_ACCOUNT_ID in GitHub Actions, or run wrangler login locally.
