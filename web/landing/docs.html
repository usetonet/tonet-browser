Documentación

Guías breves para usuarios y mantenedores del proyecto.

Instalación

Consulta la sección de descargas. Windows ofrece MSI y EXE; Linux admite .deb y script de usuario; macOS, compilación local hasta que haya bundle firmado.

Actualizaciones en el navegador

Tonet consulta la API de GitHub Releases (sin instalar binarios por ti). Por defecto comprueba al iniciar. En Ajustes (⚙) puedes elegir:

  • Solo al iniciar — una comprobación al abrir la app.
  • Automático (24 h) — repite mientras Tonet está abierto.
  • Solo manual — únicamente con «Comprobar ahora».

Los ajustes se guardan en el directorio de configuración del sistema (por ejemplo %APPDATA%\tonet en Windows).

Firma Authenticode (Windows)

Para que SmartScreen confíe en los instaladores, firma el MSI y el EXE con un certificado de publicador de código (SHA-256) en una máquina Windows con signtool del Windows SDK. 

signtool sign /fd SHA256 /a /tr http://timestamp.digicert.com /td SHA256 Tonet-x.x.x-x64.msi
signtool sign /fd SHA256 /a /tr http://timestamp.digicert.com /td SHA256 Tonet-Setup-x.x.x-x64.exe

En CI, guarda el PFX en un secreto cifrado y usa un runner auto-hospedado o una acción que invoque signtool. No subas certificados al repositorio.

Paquete Debian (.deb)

El script packaging/debian/build-deb.sh genera un .deb desde cargo build --release -p tonet. Requiere dpkg-deb (Linux). 

cargo build --release -p tonet
./packaging/debian/build-deb.sh

AppImage (Linux)

Para distribuir un AppImage universal necesitas empaquetar dependencias de GTK/GL — recomendamos evaluar linuxdeploy + plugin GTK cuando el binario esté estabilizado. El repositorio puede añadir un workflow dedicado en una release futura.

Desplegar esta landing (Cloudflare Workers)

En el directorio web/landing:

npm ci
npm run deploy

Configura CLOUDFLARE_API_TOKEN y CLOUDFLARE_ACCOUNT_ID en GitHub Actions o ejecuta wrangler login en local.
