Installation

See the downloads section. The landing reads version.json (from crates/tonet) and points primary buttons at real filenames on GitHub releases/latest/download (for example the Inno EXE and .deb). MSI and tag-scoped links use releases/download/vX.Y.Z/…. The optional TonetSetup web installer is attached when the release workflow publishes it.

In-browser updates

Tonet checks your update manifest endpoint (it does not install binaries for you). By default it checks on startup. In Settings (⚙) you can choose:

  • On startup only — one check when the app opens.
  • Automatic (24 h) — repeats while Tonet is running.
  • Manual only — only with “Check now”.

Settings are stored in the system config directory (for example %APPDATA%\tonet on Windows).

Authenticode signing (Windows)

For SmartScreen trust, sign MSI and EXE with a code-signing certificate (SHA-256) on Windows using signtool from the Windows SDK. 

signtool sign /fd SHA256 /a /tr http://timestamp.digicert.com /td SHA256 Tonet-x.x.x-x64.msi
signtool sign /fd SHA256 /a /tr http://timestamp.digicert.com /td SHA256 Tonet-Setup-x.x.x-x64.exe

In CI, store the PFX in an encrypted secret and use a self-hosted runner or an action that invokes signtool. Never commit certificates to the repository.

Debian package (.deb)

The script packaging/debian/build-deb.sh builds a .deb from cargo build --release -p tonet. Requires dpkg-deb (Linux). 

cargo build --release -p tonet
./packaging/debian/build-deb.sh

AppImage (Linux)

Shipping a universal AppImage requires bundling GTK/GL dependencies — consider linuxdeploy + the GTK plugin once the binary stabilizes. A dedicated workflow may land in a future release.

Deploy this landing (Cloudflare Workers)

From the web/landing directory:

npm ci
npm run deploy

Set CLOUDFLARE_API_TOKEN and CLOUDFLARE_ACCOUNT_ID in GitHub Actions, or run wrangler login locally.

CDN configuration (required variables)

To stop depending on GitHub releases endpoints, define CDN URLs in the landing build and in Tonet runtime. 

# web/landing/.env(.production)
TONET_CDN_BASE_URL=https://downloads.example.com
TONET_CDN_VERSION_PATH=/version.json
TONET_SITE_DOWNLOADS_PAGE=https://downloads.example.com/

# rust build environment for Tonet app updates
TONET_UPDATE_MANIFEST_URL=https://downloads.example.com/version.json
TONET_DOWNLOADS_PAGE_URL=https://downloads.example.com/

Use cases and audience

  • Minimal browsing environments where deterministic behavior matters.
  • Teams that need transparent browser internals and release control.
  • Operators that prefer CDN-governed artifacts and update manifests.
  • Power users who want a focused browser without broad platform noise.

Implementation and future work

  • Current: Servo integration foundation, policy modules, and CI coverage.
  • Next: broader web compatibility layers and stronger packaging parity.
  • Planned: deeper conformance targets, perf budgets, and expanded platform support.
