Public handbook

This page centralizes expanded project documentation: use cases, utility references, product evolution, and future implementation tracks.

Use cases

  • Low-overhead browsing in controlled environments.
  • Educational and R&D contexts where browser internals must stay transparent.
  • Operational deployment where CDN-based release distribution is required.
  • Security-conscious setups that prefer explicit policy-driven behavior.

Utilities and internal pages

  • tonet://settings: language, theme, update policy, system toggles.
  • tonet://downloads: page fetch and local save activity.
  • tonet://history: visit timeline and data management hooks.
  • tonet://new-tab: shortcuts and search integration.

Recent changes

Recent work focuses on Servo integration foundations, improved policy surfaces, update flow hardening, and stronger CI/document reproducibility. Full version notes are tracked in the changelog section after each release.

Future implementation tracks

  • Broader selector/layout support for improved compatibility.
  • Expanded cross-platform packaging and installer parity.
  • More complete test matrices for conformance and regressions.
  • Incremental performance and memory budget enforcement.

Comparison notes

Tonet is intentionally narrower than mature browsers like Chrome and Brave. It targets clarity, policy control, and operational determinism first. See Compare for a concise positioning summary.
