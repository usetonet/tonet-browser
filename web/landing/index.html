Built to get to the point

An honest MVP: network + parser + minimal renderer. Tonet does not try to render today’s industrial web; it tries to make reading and search feel human again.

Purity filter

Pages over 1 MB are rejected. Tonet only loads what fits a lightweight standard.

Custom parser

Manual extraction of title and essential text — without dragging in a full HTML engine.

Updates on your terms

Check on launch, every 24 hours, or only when you want. One click opens official downloads.

Open source

Full transparency on GitHub. Signed installers and documented pipelines for operators.

In-house engine path

Servo-based rendering is embedded for web pages — not Chromium, WebKit, or CEF. Fewer opaque layers between your policy and what hits the network.

Strict resource bounds

The 1 MB page ceiling keeps accidental megabyte payloads out of your session; failures are explicit instead of grinding the UI.

Operational transparency

Own your CDN and version.json: updates don’t depend on calling GitHub’s release API from the client at runtime.

Privacy-minded defaults

Minimal surface area for trackers and third-party SDKs compared to mainstream browsers built for maximal compatibility.

What makes Tonet different

Built for users and teams that want explicit control over what runs in the browser, how updates are delivered, and how much complexity enters the runtime.

No black box engine

Tonet keeps the engine surface understandable and auditable for your team, with the roadmap and quality gates published in the project docs.

Predictable update flow

Installer binaries and update manifests are served from your own Cloudflare-backed CDN, so operations do not depend on a third-party release API.

Docs-first project

Full public documentation includes setup guides, architecture, use cases, comparisons, release notes, and implementation plans.

Build from the repository

git clone https://github.com/usetonet/tonet-browser.git
cd tonet-browser
cargo build --release -p tonet
./target/release/tonet

Per-user install (desktop + binary)

cargo build --release -p tonet
chmod +x scripts/install-linux-user.sh
./scripts/install-linux-user.sh
